# MVP1 계획: 체크리스트 입력·저장·조회

> 상태: 계획 (착수 대상)
> 관련 문서: [체크리스트 관리 SRS](../srs/checklist-management-srs.md), [ADR 0001](../adr/0001-api-db-stack.md), [MVP2 계획](./mvp2.md)
> 실행용 상세 체크리스트: [MVP1 구현 계획](./mvp1-implementation.md)

## 1. 목표

현풍닭칼국수 매장(광화문·서면)에서 운영/식자재/공산품 체크리스트를 **지점·영업일별로 입력·저장·조회·수정**할 수 있는 최소 동작 제품을 만든다. 프론트엔드 단일 앱(현 `src/main.jsx`)을 백엔드 분리 구조로 확장한다.

## 2. 범위 (In Scope)

| 영역 | 내용 |
|---|---|
| 진입 인증 | 공용 비밀번호 1개로 진입 (SRS §5) |
| 지점·날짜 선택 | 광화문/서면, 영업일(KST) 선택 (SRS §4) |
| 운영 체크리스트 | 12종, 항목 boolean 체크 + 총점수 + 담당자명/포지션 입력·저장·조회·수정 (SRS §6) |
| 식자재 체크리스트 | 컬럼 입력(재고 포함), 차이 자동계산, 전회차 단가 자동조회, 단가 방향 화살표 + 상승 집계 표시, 담당자명 (SRS §7) |
| 공산품 체크리스트 | 단위 선택, 재고·입고 입력, 여유재고 표시, 담당자명 (1주 평균 사용량은 MVP2 통계) (SRS §8) |
| 설정 페이지 | 식자재/공산품 탭 분리, 카테고리·품목 CRUD, 품목 디폴트값, **단위 옵션 값 CRUD(식자재)**, **드래그앤드랍 순서변경** (SRS §9) |
| 저장/동시성 | 체크리스트별(14개) 저장 버튼, 빈값 저장 전 1회 경고, optimistic locking 충돌 안내 (SRS §11) |
| 이력 | 과거 영업일 조회·수정 (SRS §12) |
| 반응형 | 태블릿/모바일/데스크탑 대응 (SRS §13) |

## 3. 범위 밖 (Out of Scope → MVP2 이후)

- 월별 통계/리포트, 단가·사용량 추이 (→ [MVP2](./mvp2.md))
- 개별 계정 로그인, 역할 권한, 지점별 접근 제한
- 감사 로그(변경 이력 추적)
- 오프라인 사용
- 외부 시스템 연동

## 4. 아키텍처 / 기술 스택

[ADR 0001](../adr/0001-api-db-stack.md)을 따른다.

| 영역 | 선택 |
|---|---|
| 프론트엔드 | React/Vite (`apps/web`) |
| 백엔드 | TypeScript API (`apps/api`) |
| ORM / 마이그레이션 | Drizzle ORM / Drizzle Kit |
| DB | Turso/libSQL |
| 배포 | Vercel |

데이터 모델은 SRS §10(행 기반 + 참조 모델 + soft delete)을 따른다. 테이블 구성(11개)은 [ERD 문서](../erd.md)에서 확정했다.

## 5. 작업 분해 (마일스톤)

| # | 마일스톤 | 주요 작업 |
|---|---|---|
| M0 | 프로젝트 구조 전환 | `apps/web`, `apps/api` 모노레포 구성, Drizzle/Turso 설정, Vercel 환경변수 |
| M1 | 스키마 & 마이그레이션 | ERD 확정 → Drizzle schema → migration, 초기 시드(품목/카테고리) 적재 |
| M2 | 진입 인증 | 공용 비밀번호 진입 화면 |
| M3 | 운영 체크리스트 | 12종 입력/저장/조회/수정, 총점수 |
| M4 | 식자재 체크리스트 | 입력(판정/원인/재고 포함), 차이 자동계산, **체크리스트 진입 시 품목별 전회차 단가 즉시 표시**, 화살표/상승 집계 |
| M5 | 공산품 체크리스트 | 입력/저장/조회/수정 |
| M6 | 설정 페이지 | 식자재/공산품 탭, 카테고리·품목 CRUD, 디폴트값, 단위 옵션 CRUD, 드래그앤드랍 순서변경 |
| M7 | 동시성·검증 | optimistic locking, 빈값 경고, 충돌 안내 |
| M8 | 반응형·마감 | 반응형 점검, 백업 절차 문서화, 배포 |

## 6. 완료 기준 (Acceptance)

- 광화문/서면 각각에 대해 14종 체크리스트를 저장하고, 다른 기기에서 같은 (지점·영업일)로 조회 시 저장값이 보인다.
- 식자재 체크리스트를 열면 품목별 전회차 단가(전날 또는 최근일)가 **즉시** 채워져 보이고, 공급단가를 타이핑하면 화살표(▲빨강/▼파랑)와 우상단 상승 품목 수 색상 텍스트가 실시간 갱신된다.
- 동시 편집 시 늦게 저장한 쪽에 "다른 사람이 먼저 저장했습니다. 새로고침 후 다시 시도하세요" 안내가 뜬다.
- 품목을 추가/삭제(soft delete)해도 과거 기록 조회가 깨지지 않는다.
- 태블릿/모바일/데스크탑에서 레이아웃이 정상 동작한다.

## 7. 위험 / 선결 과제

| 항목 | 내용 |
|---|---|
| 결정 완료 | 운영 항목=코드 고정, 총점수=체크수(1점/항목), 공산품 1주 평균=MVP2 통계 (SRS §16) |
| 데이터 보존 | 백업(주 1회, 일요일) 절차 문서화 필요 (ADR 후속) |
| 전회차 단가 성능 | 품목별 최근일 조회 쿼리가 조회 800ms 목표를 넘지 않도록 인덱스 설계 |
