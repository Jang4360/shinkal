## 0-1) 확정 사항 (구현 시 반영)

- 통계 탭 1개. 사이드탭에 "통계" 추가. 상단 **[지점] + [기준 연-월]** 셀렉터 하나.
- **스냅샷 카드 = 선택 달 / 추이 카드 = 선택 달 끝점 최근 12개월**. (MVP2 §3)
- 집계는 **on-the-fly + 인덱스**(요약 테이블 미도입). 읽기 전용(쓰기 없음). (MVP2 §8)
- 차트 = **Recharts**. 색상 = 따뜻한 계열(빨강/주황/노랑/자홍), 단가 방향은 상승=빨강·하락=파랑 유지. 한 줄 **최대 3카드**.
- 1차 카드: 식자재(단가 추이 / 상승 Top5 / 사용량 합계·차이누계), 공산품(1주 평균 사용량), 운영(누락 항목 Top5 / 포지션별 월평균).
- 2차(범위 밖): 지점 비교, 운영 점수 추이, 공산품 사용량 추이, 내보내기(PDF/CSV).
- **전제**: MVP1에 `product_line.restock_qty`(입고) 적재 완료. (MVP2 §9)
- 정의 기본값: Top N=5, 추이 12개월, 단가 월값=그 달 평균(호버 최고/최저), 공산품 일별 사용량=전일재고+당일입고−당일재고(음수 0, 전일 없으면 제외).

## 1) 전제 · 준비

- [x] **입고 컬럼 선결 확인** — `product_line.restock_qty` 스키마/마이그레이션/입력 UI가 MVP1에 반영됐는지 확인. 성공기준: 공산품 체크리스트에서 입고 저장·재조회 동작. (참조: MVP2 §9)
- [x] **Recharts 의존성 추가**(`apps/web`) — 성공기준: 라인/바 차트가 렌더되고 빌드 통과. (참조: MVP2 §7)
- [x] **집계용 인덱스 점검** — `(branch_id, business_date)`, `ingredient_line(item_id)`, `operation_record(branch_id, business_date)` 등 존재 확인(대부분 MVP1에 있음). 성공기준: 월 범위 조회가 인덱스 사용. (참조: MVP2 §8, SRS §13)

## 2) 백엔드 통계 API (on-the-fly)

- [x] **에러/인증 규약 재사용** — 통계 엔드포인트도 `requireAuth` + 공통 에러 envelope 사용. 성공기준: 미인증 401, 에러 형식 동일. (참조: API §0)
- [x] `GET /api/stats/ingredients?branchId&month` — 성공기준: ①단가 추이(품목별 최근 12개월 월평균 + 월별 최고/최저), ②이번 달 단가 상승 Top5(전월 대비), ③실제 사용량 월 합계 + (실제−기준) 차이 누계 를 반환. (참조: MVP2 §4.1)
- [x] `GET /api/stats/products?branchId&month` — 성공기준: 그 달 주차별(월~일) 품목 **1주 평균 사용량** 반환. 사용량=전일재고+당일입고−당일재고(음수 0, 전일 기록 없으면 제외), 주 평균=주간 일별 사용량 평균. (참조: MVP2 §4.2, §8)
- [x] `GET /api/stats/operations?branchId&month` — 성공기준: ①자주 누락 항목 Top5(그 달 `checked=0` 빈도), ②체크리스트(12종)별 월 평균 `total_score` 반환. (참조: MVP2 §4.3)
- [x] **빈 데이터 처리** — 기록 없는 달/품목은 빈 배열/0으로 안전 반환(에러 아님). 성공기준: 데이터 없는 달 조회 시 200 + 빈 결과. (참조: MVP2 §3)

## 3) 프론트 — 통계 셸

- [x] **사이드탭 "통계" 항목 추가** + 라우트 — 성공기준: 클릭 시 통계 화면 진입, 선택 인디케이터 동작. (참조: 화면명세 S0, MVP2 §1)
- [x] **상단 [지점] + [기준 연-월] 셀렉터** — 일반 체크리스트의 영업일 대신 연-월 선택. 변경 시 통계 재조회. 성공기준: 지점/월 바꾸면 모든 카드 갱신. (참조: MVP2 §3)
- [x] **카드 그리드 레이아웃** — 한 줄 최대 3카드(4+ 금지), 큰 차트는 2칸. 따뜻한 팔레트 토큰 정의. 성공기준: 데스크탑 3열/반응형 축소, 색상 일관. (참조: MVP2 §6, UI/UX)
- [x] **Recharts 공통 차트 컴포넌트**(라인/바 + 툴팁) — 성공기준: 호버 툴팁 동작. (참조: MVP2 §7)

## 4) 프론트 — 식자재 카드

- [x] **월별 단가 추이(라인)** — 품목 선택 → 최근 12개월 월평균 단가 라인, 점 호버 시 그 달 최고/최저/평균 툴팁(치우면 사라짐). 성공기준: 품목 바꾸면 라인·툴팁 갱신. (참조: MVP2 §4.1)
- [x] **이번 달 단가 상승 Top5(막대)** — 전월 대비 상승폭 순. 성공기준: 상승 품목만 막대로, 색=빨강 계열. (참조: MVP2 §4.1)
- [x] **월별 사용량(막대/표)** — 실제 사용량 합계 + (실제−기준) 차이 누계. 성공기준: 값이 API와 일치. (참조: MVP2 §4.1)

## 5) 프론트 — 공산품 카드

- [x] **1주 평균 사용량(막대)** — 그 달 주차별(~4~5개) 막대. 성공기준: 입고 반영 공식대로 계산된 값 표시, 데이터 없으면 빈 상태. (참조: MVP2 §4.2)

## 6) 프론트 — 운영 카드

- [x] **자주 누락 항목 Top5(막대/표)** — 그 달 미체크 빈도 상위. 성공기준: 항목 텍스트 + 횟수 표시. (참조: MVP2 §4.3)
- [x] **포지션별 월 평균 점수(막대)** — 12종별 평균 total_score. 성공기준: 12개 막대, 약한 포지션 식별 가능. (참조: MVP2 §4.3)

## 7) 기능 검증 (E2E)

- [x] 지점/월 셀렉터로 광화문·서면, 여러 달 전환 시 모든 카드가 해당 기준으로 갱신.
- [x] 식자재: 한 품목의 단가 라인이 최근 12개월로 그려지고, 점 호버 시 최고/최저/평균 툴팁이 뜬다.
- [x] 식자재: 상승 Top5가 전월 대비 실제 상승 품목과 일치.
- [x] 공산품: 입고가 있던 주/없던 주의 1주 평균 사용량이 공식(전일+입고−당일)대로 계산됨. 전일 기록 없는 날 제외 확인.
- [x] 운영: 의도적으로 특정 항목을 여러 날 미체크 → 누락 Top5 상단에 노출. 포지션별 평균 점수가 저장값과 일치.
- [x] 데이터 없는 달 조회 시 빈 상태 카드(에러 없음).
- [x] 조회 응답 800ms 이내(대표 케이스). (SRS §13)
