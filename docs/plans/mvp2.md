# MVP2 계획: 통계/리포트

> 상태: 계획 (MVP1 완료 후 착수)
> 관련 문서: [SRS](../srs/checklist-management-srs.md), [ERD](../erd.md), [ADR 0001](../adr/0001-api-db-stack.md), [UI/UX](../ui_ux.md)
> 실행용 상세 체크리스트: [MVP2 구현 계획](./mvp2-implementation.md)

## 1. 배경

MVP1은 체크리스트 입력/저장/조회/수정과 품목·카테고리 관리에 집중한다. MVP2는 누적된 기록을 **월 단위로 보는 통계 탭** 하나를 제공한다. (사이드탭에 "통계" 항목 추가)

통계가 의미를 가지려면 과거 기록 보존이 전제이므로 MVP1에서 **soft delete(하드 삭제 금지) + 행 기반 + 참조 모델(이름)/입력값 동결**을 지킨다(SRS §10).

## 2. 목표 (관리자 관점)

- 식자재 **단가 변동**과 **사용량(기준 대비 차이)** 을 월 단위로 추적해 원가/발주 관리.
- 공산품 **1주 평균 사용량**(입고 반영)으로 소비 속도 파악.
- 운영 체크리스트에서 **자주 누락되는 항목**과 **약한 포지션**을 찾아 교육/프로세스 개선.

## 3. 기간 모델 (중요)

- 상단에 **[지점] + [기준 연-월]** 셀렉터 하나. 모든 카드가 이 선택을 기준으로 동작한다.
- **스냅샷형 카드**(이번 달 합계/평균/Top N): 선택한 달 데이터만.
- **추이형 카드**(단가 추이, 점수 추이 등): 선택한 달을 **끝점으로 최근 6~12개월**을 함께 표시(한 달만으론 추세가 안 보이므로). 셀렉터는 하나로 유지.

## 4. 1차 통계 카드 (이번 범위)

### 4.1 식자재

| 카드 | 유형 | 정의 / 표시 |
|---|---|---|
| 월별 단가 추이 | 추이(라인) | 품목 선택(드롭다운/검색) → 최근 6~12개월 **월평균 공급단가** 라인. 점 hover 시 그 달 **최고/최저/평균** 툴팁(치우면 사라짐). |
| 이번 달 단가 상승 Top N | 스냅샷(막대) | 전월 대비 단가가 오른 품목을 상승폭 순으로 막대. |
| 월별 사용량 | 스냅샷(막대/표) | **실제 사용량 월 합계** + **(실제−기준) 차이 누계**. (평균은 제외) |

### 4.2 공산품

| 카드 | 유형 | 정의 / 표시 |
|---|---|---|
| 1주 평균 사용량 | 스냅샷(막대) | 매주 월요일 전주(월~일) 기준 품목별 1주 평균 사용량. 그 달의 주차별(~4~5개) 막대. **사용량 = 전일 재고 + 당일 입고 − 당일 재고** (입고 인라인 컬럼 기반, §8). |

### 4.3 운영

| 카드 | 유형 | 정의 / 표시 |
|---|---|---|
| 자주 누락되는 항목 Top N | 스냅샷(막대/표) | 해당 월에 `checked=0`이었던 빈도가 높은 점검 항목 상위 N개. (가장 actionable — `operation_check` 항목 단위 집계) |
| 포지션별 월 평균 점수 | 스냅샷(막대) | 12종 체크리스트별 월 평균 `total_score`. 약한 포지션/시간대 식별. |

## 5. 2차 (후순위)

- 단가 상승 알림 요약(상세 목록), 공산품 사용량 월간 추이, 운영 **총점수 월별 추이(라인)**.
- **지점 비교**(광화문 vs 서면 나란히).
- **내보내기**(PDF/CSV). 1차는 화면 표시만.

## 6. 화면 레이아웃 / 비주얼

- **한 줄 최대 3개 카드** (4개 이상은 가독성 저하 — 금지). 큰 차트는 2칸(약 2/3 폭) 차지 가능.
- 구성: 상단 핵심 **요약 KPI 카드 줄(최대 3)** → 아래 영역별 **차트 카드**(식자재 → 공산품 → 운영 순) 그리드. 관련 지표끼리 묶고 여백/위계 확보(첨부 대시보드 레퍼런스처럼 한눈에 들어오게).
- **색상: 따뜻한 계열 위주**(빨강/주황/노랑/자홍). 차트 팔레트 예시:
  - 빨강 `#EF4444`(기본/강조), 주황 `#F97316`, 노랑 `#F59E0B`, 자홍 `#DB2777`.
  - 단, 단가 방향 일관성을 위해 **상승=빨강 / 하락=파랑(`#2563EB`)** 은 유지(SRS §7.2).
- 흰 배경·Pretendard·카드 토큰 등은 [UI/UX](../ui_ux.md) 따름.

## 7. 차트 도구

- **Recharts** 사용(React 친화·선언적, hover 툴팁 기본 내장 → 단가 최고/최저 호버에 적합). 추이=Line, 스냅샷/Top N=Bar.

## 8. 데이터 / 집계 방식

- **기본 on-the-fly 집계 + 인덱스.** 데이터가 작아 별도 요약 테이블 없이 기존 테이블에서 매 조회 계산(조회 800ms 목표, SRS §13).
- **공산품 입고 = 공산품 체크리스트의 인라인 컬럼**으로 입력(확정). 입고일에만 입력, 평소 빈칸. 직원이 매일 쓰는 화면이라 동선이 짧고 누락 위험이 낮다.
  - 스키마 영향: `product_line`에 **`restock_qty`(REAL, 입고량)** 컬럼 추가.
  - 1주 평균 사용량은 위 입고 반영 공식으로 계산. 주별 추이를 길게 쌓을 필요가 생기면 그때 주간 스냅샷 테이블(`product_weekly_usage(branch_id, week_start, item_id, avg_usage)`)을 월요일 잡으로 도입(현 시점 미도입).

## 9. 선결 조건 / MVP1 영향

- MVP1 기록이 행 기반(참조 모델 + 입력값 동결)으로 정확히 적재돼 있을 것.
- **MVP1에 입고 컬럼 추가 필요**: 공산품 체크리스트(S4)에 `입고` 인라인 컬럼, `product_line.restock_qty` 스키마/마이그레이션, 관련 문서(ERD §11, SRS §8, API §5, 화면명세 S4) 동기화. (통계의 공산품 사용량 산출 전제)
- 단가 색상 규칙 확정(SRS §7.2.1).
- 데이터 영구 보존 유지.

## 10. 범위 밖 (현 시점)

- 실시간 대시보드, 예측/추천.
- 외부 회계/발주 시스템 연동.
