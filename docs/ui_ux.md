# UI/UX 설계

> 상태: 작성 중 (리뷰 대상)
> 대상 서비스: 현풍닭칼국수 매장 운영 체크리스트 (신칼)
> 관련 문서: [SRS](./srs/checklist-management-srs.md), [MVP1 계획](./plans/mvp1.md)

## 0) 목적

기존 화면이 "AI가 찍어낸 듯한" 인상을 주는 요소(전 영역 빨강 상단 stroke, 크림색 배경, 과도한 굵기·그림자, 지나치게 큰 제목)를 걷어내고, **깨끗한 흰 배경의 운영 관리자 화면**으로 다시 설계한다. 정보 위계와 여백을 우선한다.

설계 기준:
- **레이아웃/배경**: 부산이음길 관리자(`admin.busaneumgil.com`) — 흰 배경, 절제된 카드, 명확한 사이드 내비.
- **버튼/컴포넌트 톤**: 토스증권(`tossinvest.com`) — 흑백 기반 미니멀, 둥근 모서리, 큰 탭 영역, 낮은 채도.
- **품질 원칙**: [ui-skills.com](https://www.ui-skills.com/skills/) — 타이포 위계, 일관된 간격, 대비/접근성, 불필요한 굵기·장식 제거("deslop").

## 1) 핵심 변경 (이번 적용)

| # | 변경 | Before | After |
|---|---|---|---|
| 1 | 배경 | 크림색(`#fffdf8`/`#f7f5ef`) | **흰색**(`#ffffff`) |
| 2 | 상단 빨강 stroke | `border-top: 4px solid red` (topbar/hero/inspector) | **제거** |
| 3 | 폰트 | Inter | **Pretendard** |
| 4 | 체크리스트 제목 | `매장 오픈 [홀]` 최대 58px | **20px**로 축소, 일관화 |
| 5 | 글자 굵기 | 800~950 남발 | 600/700 중심 |
| 6 | 테두리 색 | 전부 빨강 틴트 | **중립 회색**, 빨강은 강조에만 |
| 7 | 그림자 | `0 14px 40px` 등 과함 | 얕고 은은하게 |
| 8 | 버튼 | 각지고 과한 굵기 | 토스식 둥근 모서리·넉넉한 탭 |

## 2) 디자인 토큰

### 색상

| 토큰 | 값 | 용도 |
|---|---|---|
| `--bg` | `#ffffff` | 페이지 배경 |
| `--surface` | `#ffffff` | 카드/패널 |
| `--surface-2` | `#f7f8fa` | 옅은 강조 영역(섹션 헤더 등) |
| `--surface-3` | `#e7e9ee` | 선택된 사이드탭 배경 |
| `--border` | `#e5e7eb` | 기본 테두리 |
| `--border-strong` | `#d1d5db` | 입력/구분선 강조 |
| `--text` | `#1a1a1a` | 본문 |
| `--text-2` | `#6b7280` | 보조 텍스트/라벨 |
| `--text-3` | `#9ca3af` | 비활성/캡션 |
| `--brand` | `#EF4444` | **강조 전용**(주요 버튼, 활성 탭, 체크 상태, 점수 링, 단계 배지) |
| `--brand-dark` | `#C1121F` | 강조 hover/press |
| `--up` | `#EF4444` | 단가 상승 텍스트 |
| `--down` | `#2563EB` | 단가 하락 텍스트 |

> 빨강은 더 이상 "테두리 기본색"이 아니다. 면/선의 기본은 회색, 빨강은 **상태·강조**에만 쓴다.

### 타이포그래피 (Pretendard)

| 토큰 | 크기/굵기 | 용도 |
|---|---|---|
| `--text-title` | 20px / 600 | 페이지·체크리스트 제목 (구 58px) |
| `--text-h2` | 16px / 600 | 섹션 제목, 패널 제목 |
| `--text-body` | 15px / 500 | 점검 항목, 본문 |
| `--text-label` | 13px / 600 | 라벨, 메타 키 |
| `--text-caption` | 12px / 500 | 캡션, 보조 |
| 점수 숫자 | 40px / 700 | 점수 링 중앙 숫자(데이터 강조라 예외) |

- 기본 `line-height` 본문 1.5, 제목 1.3.
- 굵기는 **400/500/600/700만** 사용. 800/900 금지(AI 인상 제거).

### 간격·모서리·그림자

| 토큰 | 값 |
|---|---|
| 간격 스케일 | 4 · 8 · 12 · 16 · 20 · 24 · 32 (4px 배수) |
| `--radius-sm` | 8px (입력·작은 칩) |
| `--radius` | 12px (카드·버튼) |
| `--radius-lg` | 16px (큰 패널) |
| `--radius-pill` | 999px (배지) |
| `--shadow-sm` | `0 1px 2px rgba(0,0,0,.05)` |
| `--shadow` | `0 1px 3px rgba(0,0,0,.08)` |

> 백드롭 블러·큰 드롭섀도는 제거한다. 카드 분리는 **얇은 회색 테두리 + 얕은 그림자**로 한다.

## 3) 컴포넌트 가이드

### 버튼 (토스식)

| 종류 | 스타일 |
|---|---|
| Primary | 채움 배경(`--brand`), 흰 글자, `--radius`(12px), 높이 48~52px, 굵기 600, hover 시 `--brand-dark`, press 시 살짝 축소(`scale .98`) |
| Secondary | 흰 배경, `--border` 테두리, `--text` 글자, 같은 radius/높이 |
| Subtle/Icon | 배경 없음/`--surface-2`, 회색 글자, hover 시 옅은 배경 |
| Danger(초기화) | Secondary 위에 글자/아이콘만 `--brand` |

- 탭 영역은 최소 44px(터치). 모서리 일관(12px). 그림자 대신 색 대비로 위계.

### 사이드탭 (운영 12종 / 식자재 / 공산품 / 설정)

- 좌측 세로 내비. 항목은 좌측 정렬 텍스트 + 선택 시 `--surface-3` 배경으로만 구분한다. 빨간 좌측 인디케이터는 쓰지 않는다.
- 단계(오픈/운영/마감)는 소제목(`--text-3`)으로 그룹핑.
- 모바일: 상단 가로 스크롤 탭 또는 드로어로 전환.

### 상단 컨텍스트 바

- 카드처럼 떠 보이지 않게 본문 상단 좌우 끝까지 꽉 차는 sticky 바를 사용한다.
- 테두리/라운드/그림자는 제거하고 하단 `--border` 1px 구분선만 둔다.
- 지점/영업일 컨트롤은 우측 정렬한다.

### 카드/패널

- `--surface` + `--border` 1px + `--radius` + `--shadow-sm`. **상단 강조선 없음.**
- 내부 패딩 16~20px.

### 체크리스트 제목 영역(구 page-hero)

- 단계 배지(pill, `--brand`) + 제목 20px/600 한 줄. 큰 히어로 텍스트 제거.
- 부제(`subtitle`)는 `--text-2` 14px.

### 점검 항목 행 (check-row)

- 체크박스(20~24px, 체크 시 `--brand` 채움) + 번호(`--text-3`) + 항목 텍스트 15px/500.
- hover 옅은 회색, 체크 시 옅은 회색 배경 + 취소선은 색만 흐리게(과한 두께 금지).

### 점수 링 (inspector)

- 흰 배경 카드, 링은 `conic-gradient(--brand, --border)`. 중앙 숫자 40px/700, 하단 "N점" 캡션.
- "점검 필요/양호" 라벨 16px/600 + 보조 문구(`날짜 미입력 · 담당자 미입력`)는 `--text-2` 13px.

### 식자재 표 요소

- 단가 상태: 상승 `--up`(빨강) / 하락 `--down`(파랑) / 동일 회색 텍스트. 화살표나 점 아이콘은 쓰지 않는다(SRS §7.2).
- 우상단 상승 집계 텍스트 색: 1 초록 / 2~3 노랑 / 4↑ 빨강 (SRS §7.2.1).
- 판정 라디오(정상/과다/부족)는 세그먼트 형태, 선택 시 `--brand` 강조.

## 4) 레이아웃

- 데스크탑: 좌 사이드탭 + 본문(체크리스트) + 우 인스펙터(점수/누락) 3열, 본문 max-width ~1100px.
- 태블릿: 인스펙터를 본문 아래로 접기.
- 모바일: 단일 컬럼, 사이드탭은 상단 탭/드로어.
- 전 영역 흰 배경, 카드 간 간격 16px.

## 5) 구현 매핑 (styles.css / index.html)

| 위치 | 조치 |
|---|---|
| `index.html` | Pretendard 웹폰트 `<link>` 추가 |
| `:root` | `font-family: Pretendard …`, 배경 `#fff`, 위 토큰 정의 |
| `.app` | 배경 `#fff` |
| `.topbar` / `.page-hero` / `.inspector-panel` | `border-top` 빨강 제거, 테두리 `--border`, 그림자 약화 |
| `.page-hero h1` | `font-size: 20px; font-weight: 600; line-height: 1.3` |
| 빨강 틴트 테두리 | `--border`(회색)로 교체, 빨강은 배지/활성/체크/링에만 |
| `font-weight: 800/900/950` | 600/700로 정리 |
| 버튼류 | `--radius`(12px), 굵기 600/700, press 피드백 |

## 6) 접근성

- 본문 대비 AA 이상(텍스트 `#1a1a1a` on `#fff`).
- 포커스 링 유지(`outline`), 단 색은 `--brand` 24% 알파로 은은하게.
- 터치 타깃 ≥ 44px, 색에만 의존하지 않게 단가 상태는 텍스트(상승/하락/동일)를 함께 표시.
