# 화면 명세

> 상태: 작성 중 (리뷰 대상 — 특히 **버튼 구성**은 합의 필요)
> 대상 서비스: 현풍닭칼국수 매장 운영 체크리스트 (신칼)
> 관련 문서: [SRS](./srs/checklist-management-srs.md), [ERD](./erd.md), [API](./api.md), [UI/UX](./ui_ux.md)

## 0) 공통 규칙

### 화면 목록

| # | 화면 | 경로(예시) |
|---|---|---|
| S1 | 진입 인증 | `/login` |
| S0 | 공통 셸(사이드탭 + 상단바) | 모든 화면 공통 |
| S2 | 운영 체크리스트 (12종) | `/operations/:checklistId` |
| S3 | 식자재 체크리스트 | `/ingredients` |
| S4 | 공산품 체크리스트 | `/products` |
| S5 | 설정 — 식자재 탭 | `/settings/ingredients` |
| S6 | 설정 — 공산품 탭 | `/settings/products` |

### 버튼/액션 표기 규약

각 화면의 액션은 아래 형식으로 적는다.

| 버튼/액션 | 위치 | 동작 | 결과/이동 | 관련 API | 노출 조건 |
|---|---|---|---|---|---|

- **저장 정책**: 체크리스트는 **별도 "수정 버튼"이 없다.** 과거 기록도 불러와 **즉시 인플레이스 수정**하고, 변경이 감지되면 **저장 버튼이 활성화**된다(변경 없으면 비활성). 저장 시 version/카탈로그 버전 검증(SRS §11, §12).
- **이탈 가드**: 미저장 변경이 있는 채로 화면/지점/날짜 이동·로그아웃 시 모달("수정사항을 저장하지 않고 이동할까요?") + [저장]/[저장 안 함]/[취소] (S0 참고).
- **삭제 정책**: 체크리스트 **기록 삭제는 없다**(영구 보존, SRS §13). 삭제(soft delete)는 설정의 마스터(카테고리/품목/단위옵션)에만 있다.
- **초기화**: 현재 화면의 입력만 비우는 로컬 동작(저장 안 함). 저장된 기록에는 영향 없음.

## S1) 진입 인증 (랜딩/스플래시)

공용 비밀번호 1개로 진입한다(SRS §5). 이 화면은 **서비스 랜딩(스플래시) 역할**을 겸한다.

### 레이아웃

- 화면 **중앙 정렬**, 흰 배경.
- 상단: **신칼 로고(크게)** — 일반 웹사이트 로딩/스플래시 화면에서 로고 보여주는 정도 크기(대략 96~128px 높이 권장).
- 로고 아래: **서비스 명 텍스트(크게)** — 예: "신칼 체크리스트".
- 그 아래: 비밀번호 입력 + 진입 버튼(세로 스택, 너비 ~320px).

### 구성요소

| 요소 | 설명 |
|---|---|
| 로고 | 신칼 로고(대형) |
| 타이틀 | 서비스명 대형 텍스트 |
| 비밀번호 입력 | 공용 비밀번호 입력 필드(마스킹) |
| 표시 토글 | 비밀번호 보기/숨기기 |

### 액션

| 버튼/액션 | 위치 | 동작 | 결과/이동 | 관련 API | 노출 조건 |
|---|---|---|---|---|---|
| 진입 | 입력 하단(주요 버튼) | 비밀번호 검증 | 성공 시 토큰 저장 후 마지막/기본 화면으로 | `POST /auth/login` | 항상 |
| 표시 토글 | 입력 우측 | 마스킹 on/off | — | — | 항상 |

### 상태/검증

- 비밀번호 불일치 → `INVALID_PASSWORD`(401) **인라인 에러**("비밀번호가 올바르지 않습니다"), 입력 유지.
- 토큰 만료/없음으로 보호 화면 진입 시 이 화면으로 리다이렉트.
- 비밀번호는 서버 환경변수(`APP_PASSWORD`)로 관리(코드 하드코딩·DB 저장 안 함). 토큰은 단일 토큰(refresh 없음), httpOnly 쿠키 권장.

## S0) 공통 셸 (사이드탭 + 상단바)

모든 보호 화면을 감싸는 레이아웃. **좌측 사이드탭은 상시 고정 노출**, 상단에 지점/날짜.

### 사이드탭 구조 (상시 노출)

```
[ 신칼 로고 ]
─────────────
운영
  └ 매장 오픈 [홀]
  └ 매장 오픈 [주방 서브]
  └ 매장 오픈 [주방 메인]
  └ 매장 운영 [홀 - 메인]
  └ … (총 12개, 들여쓰기된 하위 리스트)
  └ 매장 마감 [주방 서브]
식자재
공산품
설정
─────────────
(하단 고정) 로그아웃
```

- **운영** 아래에 **12종이 들여쓰기된 하위 리스트**로 펼쳐져 상시 표시된다(오픈/운영/마감 순).
- 선택 항목은 빨간 선 없이 더 진한 회색 배경(`--surface-3`)으로만 표시한다(UI/UX §3).
- **로그아웃 버튼은 사이드탭 최하단에 고정**(관리자 페이지 스타일).

### 구성요소

| 요소 | 설명 |
|---|---|
| 사이드탭 | 운영(+12 하위) · 식자재 · 공산품 · 설정 + 하단 로그아웃 |
| 상단바 | 본문 상단에 좌우 끝까지 꽉 차는 sticky 바. 하단 1px 구분선만 사용 |
| 지점 선택 | 광화문 / 서면 드롭다운(상단바 우측) |
| 영업일 선택 | 날짜 선택(KST). 날짜 입력 칸 전체 클릭 시 달력 열림 |
| 브랜드/로고 | 사이드탭 상단 |

### 액션

| 버튼/액션 | 위치 | 동작 | 결과/이동 | 관련 API | 노출 조건 |
|---|---|---|---|---|---|
| 사이드탭 항목 | 좌측 | 화면 전환 | 해당 체크리스트/설정으로 | (이동 후 화면별 GET) | 항상 |
| 운영 하위(12) | 좌측 들여쓰기 | 해당 운영 체크리스트로 | `/operations/:checklistId` | (GET /operations) | 항상 |
| 지점 변경 | 상단 | 지점 컨텍스트 변경 | 현재 화면 데이터 재조회 | 화면별 GET | 항상 |
| 영업일 변경 | 상단 | 영업일 변경 | 현재 화면 데이터 재조회 | 화면별 GET | 체크리스트 화면 |
| 지점 목록 로드 | — | 초기 1회 | 드롭다운 채움 | `GET /branches` | 진입 시 |
| **로그아웃** | 사이드탭 최하단 | 토큰 폐기(클라이언트) | **S1 비밀번호 화면으로 이동** | — (stateless) | 항상 |

> 화면 전환/지점/날짜 변경 시 **저장하지 않은 변경이 있으면** 이탈 경고 모달을 띄운다(아래 "이탈 가드" 참고).

### 이탈 가드 (저장 안 한 변경 보호)

- 체크리스트에 **변경 감지된 미저장 입력**이 있는 상태에서 다른 화면/지점/날짜/로그아웃으로 이동하려 하면 모달을 띄운다.
- 모달 문구: "수정사항을 저장하지 않고 이동할까요?"
- 버튼: **[저장]**(저장 후 이동) · **[저장 안 함]**(버리고 이동) · [취소](그대로 머무름).

## S2) 운영 체크리스트 (12종)

12종이 같은 화면을 공유하고 `checklistId`로 구분(SRS §6). 항목은 코드 고정.

### 구성요소

| 요소 | 설명 |
|---|---|
| 제목 영역 | 단계 배지(오픈/운영/마감) + 체크리스트명(20px) |
| 메타 스트립 | 지점 · 영업일 · 담당자 · 점수 |
| 담당자 입력 | 담당자명 + 템플릿 `roles` 기준 담당 포지션 입력칸(담당자명 포함 최대 3칸) |
| 섹션/항목 | 섹션별 점검 항목(체크 토글, boolean) |
| 인스펙터 | 점수 링(체크수/전체) + 누락 항목 목록 |

### 액션

| 버튼/액션 | 위치 | 동작 | 결과/이동 | 관련 API | 노출 조건 |
|---|---|---|---|---|---|
| 항목 체크 토글 | 항목 행 | 체크/해제(즉시 반영) | 점수 링 실시간 갱신, 저장 버튼 활성화 | — | 항상 |
| 저장 | 하단/인스펙터 | 기록 저장(신규=INSERT, 기존=version 검증 UPDATE) | 성공 토스트 / 충돌 안내 | `PUT /operations` | **변경 감지 시 활성** |
| 초기화 | 상단 우측 | 현재 화면 입력 비우기(로컬) | 확인 후 비움 | — | 항상 |
| PDF 내보내기 | 인스펙터 | 현재 체크리스트 PDF 출력 | 파일 다운로드 | (프론트 생성) | 항상 |

### 상태/검증

- 총점수 = 체크 수(1점/항목), 서버 계산(SRS §6.2).
- 빈 값(담당자 등) 상태로 저장 시 **1회 경고 후 진행**(SRS §11).
- `409 VERSION_CONFLICT` → "다른 사람이 먼저 저장했습니다. 새로고침 후 다시 시도하세요".
- 운영은 카탈로그 의존이 없어 `CATALOG_CHANGED` 없음.

## S3) 식자재 체크리스트

활성 품목 전체가 행으로 표시되고, 전회차 단가는 진입 시 즉시 표시(SRS §7, API §4).

### 구성요소 (표 컬럼)

| 컬럼 | 입력/표시 |
|---|---|
| 품목 | 표시(마스터 JOIN). 카테고리는 반복 텍스트가 아니라 카테고리 변경 지점의 섹션 헤더 행으로 표시 |
| 단위 / 단위 옵션 | 선택(그날 값) |
| 기준 사용량 / 실제 사용량 | 입력 |
| 차이(실제-기준) | 자동 계산 |
| 재고 | 입력 |
| 공급단가 | 입력 |
| 전회차 단가 | 표시(자동 조회) |
| 단가 | 자동 텍스트: 상승/하락/동일, 전회차 없으면 `-` |
| 판정 | 라디오: 부족/정상/과다 표시 순서 (필수, 저장값 후보는 정상/과다/부족 유지) |
| 원인 | 텍스트(선택) |
| 상승 집계 | 표 우상단 텍스트(1초록/2~3노랑/4↑빨강) |
| 담당자명 | 입력 |

### 액션

| 버튼/액션 | 위치 | 동작 | 결과/이동 | 관련 API | 노출 조건 |
|---|---|---|---|---|---|
| 값 입력 | 표 셀 | 사용량/단가/재고/판정/원인 입력(즉시) | 차이·단가 텍스트·집계 실시간 갱신, 저장 버튼 활성화 | — | 항상 |
| 저장 | 하단 | 기록 저장(version + catalogVersion 검증) | 성공 / 충돌 안내 | `PUT /ingredients` | **변경 감지 시 활성** |
| 초기화 | 상단 우측 | 현재 입력 비우기(로컬) | 확인 후 비움 | — | 항상 |
| PDF 내보내기 | 상단/하단 | 표 PDF 출력 | 다운로드 | (프론트 생성) | 항상 |

> 품목 **행 추가/삭제 버튼은 없다.** 품목은 설정(S5)에서 관리하고 체크리스트는 활성 품목을 자동 표시한다.

### 상태/검증

- 판정 미선택 방지(필수). 빈 수치 저장 시 1회 경고.
- `409 VERSION_CONFLICT`(기록 충돌) / `409 CATALOG_CHANGED`(그 사이 설정 변경 → 설정 우선, 새로고침 안내).
- 전회차 단가는 읽기 전용(서버가 직전 기록에서 조회/스냅샷).

## S4) 공산품 체크리스트

활성 품목 전체가 행으로 표시(SRS §8, API §5).

### 구성요소 (표 컬럼)

| 컬럼 | 입력/표시 |
|---|---|
| 품목 | 표시(마스터 JOIN) |
| 단위 | 선택(박스/봉/묶음/줄/짝/개/통). 신규 행 기본값은 품목 기본 단위 |
| 입고 | 입력(입고가 있는 날만, 평소 빈칸). MVP2 통계 사용량 산출용 |
| 재고 | 입력 |
| 여유재고 | 표시(품목 마스터 기준값, 읽기 전용) |
| 담당자명 | 입력 |

> 표시 순서: 품목 - 단위 - 입고 - 재고 - 여유재고.

### 액션

| 버튼/액션 | 위치 | 동작 | 결과/이동 | 관련 API | 노출 조건 |
|---|---|---|---|---|---|
| 값 입력 | 표 셀 | 단위 선택·재고·입고 입력(즉시) | 저장 버튼 활성화 | — | 항상 |
| 저장 | 하단 | 기록 저장(version + catalogVersion) | 성공 / 충돌 안내 | `PUT /products` | **변경 감지 시 활성** |
| 초기화 | 상단 우측 | 현재 입력 비우기(로컬) | 확인 후 비움 | — | 항상 |
| PDF 내보내기 | 상단/하단 | 표 PDF 출력 | 다운로드 | (프론트 생성) | 항상 |

> 1주 평균 사용량은 입력하지 않는다(MVP2 통계).

## S5) 설정 — 식자재 탭

카테고리 / 품목 / 단위 옵션 관리(SRS §9.1). 순서변경은 드래그앤드랍.

### 편집 패턴 (인라인 — 첨부 관리자 UI 참고)

설정은 별도 모달 폼이 아니라 **인라인 편집**으로 한다. 각 관리 대상(카테고리/품목/단위옵션)은 **카드** 하나로 묶고, 카드 안에 **리스트(row 표)** 가 있다.

- 카드 **우측 상단에 `+ 추가` 버튼**. 누르면 리스트 **맨 아래로 자동 스크롤**되며 빈 입력 행이 추가되고 사용자가 바로 입력한다.
- 각 row **오른쪽에 연필(수정) 아이콘 + 휴지통(삭제) 아이콘**.
- 연필 클릭 → 그 행이 **인라인 편집 상태**(셀이 입력 가능)로 전환, 확인 시 저장.
- 휴지통 클릭 → **삭제 확인 모달** 후 soft delete.
- 행 왼쪽 드래그 핸들로 순서변경.
- 빈 입력으로 추가/저장 시 인라인 검증 에러.

### 구성요소

| 영역 | 설명 |
|---|---|
| 탭바 | 식자재 / 공산품 |
| 카테고리 목록 | 행: 카테고리명, 드래그 핸들, 수정/삭제 |
| 품목 목록 | 카테고리별 그룹. 행: 품목명, 기본 단위/옵션/기준사용량, 드래그 핸들, 수정/삭제 |
| 단위 옵션 목록 | 단위(Kg/g/구)별. 행: 옵션 값, 드래그 핸들, 수정/삭제 |

### 액션 — 카테고리

| 버튼/액션 | 동작 | 관련 API |
|---|---|---|
| 추가 | 새 카테고리 입력(이름) | `POST /settings/ingredient-categories` |
| 수정 | 이름 변경(행 version) | `PATCH /…/{categoryId}` |
| 삭제 | soft delete(확인 모달) | `DELETE /…/{categoryId}` |
| 순서변경 | 드래그앤드랍 | `PATCH /…/reorder` |

### 액션 — 품목

| 버튼/액션 | 동작 | 관련 API |
|---|---|---|
| 추가 | 품목 입력(이름·소속 카테고리·기본단위/옵션/기준사용량) | `POST /settings/ingredient-items` |
| 수정 | 위 항목 변경(행 version) | `PATCH /…/{itemId}` |
| 삭제 | soft delete(확인 모달) | `DELETE /…/{itemId}` |
| 순서변경 | 드래그앤드랍(카테고리 내) | `PATCH /…/reorder` |

### 액션 — 단위 옵션

| 버튼/액션 | 동작 | 관련 API |
|---|---|---|
| 추가 | 단위 선택 + 옵션 값 입력 | `POST /settings/ingredient-unit-options` |
| 수정 | 값 변경(행 version) | `PATCH /…/{optionId}` |
| 삭제 | soft delete(확인 모달) | `DELETE /…/{optionId}` |
| 순서변경 | 드래그앤드랍(단위 내) | `PATCH /…/reorder` |

### 상태/검증

- 모든 변경은 식자재 **카탈로그 버전 +1**(설정↔체크리스트 충돌 감지, SRS §11).
- 행 `version` 충돌 → `409 VERSION_CONFLICT`.
- 같은 단위에 같은 옵션 값 → `409 DUPLICATE`.
- 삭제는 soft delete라 과거 기록 표시에 영향 없음(참조 모델).
